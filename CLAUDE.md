# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## What This Project Does

Fetches top Hacker News stories (default 30), filters noise (Ask/Tell/low-score/Show HN without URL), keeps the top N by score (default 10), formats a ranked markdown digest, and writes it to `output/digest-YYYY-MM-DD.md`. Timezone is hardcoded to `Asia/Ho_Chi_Minh` (ICT). Fetch count, keep count, output date, and dry-run mode are all configurable via CLI flags.

## Commands

```bash
node generate.js                        # Generate today's digest
node generate.js --date 2026-06-30      # Override the date in filename/header
node generate.js --top 50 --keep 15     # Fetch 50 stories, keep top 15
node generate.js --dry-run              # Print to stdout instead of writing file
```

All flags can be combined. Requires Node 18+ for built-in `fetch`.

There are no tests, no build step, no lint step, and no external dependencies. The `package.json` test script is a stub.

## Architecture

This is a single-file project (`generate.js`, ~197 lines). Everything runs in one `main()` call:

1. **Parse args** — `parseArgs()` reads CLI flags (`--date`, `--top`, `--keep`, `--dry-run`) with validation.
2. **Fetch** — pulls top N story IDs from HN Firebase API, then fetches each story in parallel via `Promise.all`. Individual fetch failures return `null` and are skipped (non-fatal).
3. **Filter** — `filterStories()` applies the same rules as `.claude/agents/content-filter.md`: story type only, drop Ask/Tell/Who-is-hiring prefixes, Show HN needs URL + score ≥ 100, score ≥ 50 floor, must have URL, sort by score, take top N.
4. **Format** — `formatDigest()` builds markdown following `.claude/skills/digest-format/SKILL.md`: H1 title, subtitle with count + time, horizontal rule, ranked entries with score/comments/author metadata and a contextual summary, footer with ISO timestamp.
5. **Write** — saves to `output/digest-YYYY-MM-DD.md`, or prints to stdout if `--dry-run`.

The `summarize()` function generates a templated sentence from the story's domain, title, score, and comment count — it is not AI-generated.

## Key Defaults (generate.js)

- `HN_BASE` — `https://hacker-news.firebaseio.com/v0`
- `TIMEZONE` — `Asia/Ho_Chi_Minh`
- `DEFAULT_TOP_N_FETCH` — 30 (overridable with `--top`)
- `DEFAULT_TOP_N_KEEP` — 10 (overridable with `--keep`)

## Claude Code Configuration

- **MCP server**: `filesystem` via `.mcp.json` (scoped to project root)
- **Skill**: `.claude/skills/digest-format/SKILL.md` — defines the output format, tone (neutral, factual, no hype), and edge cases
- **Agent**: `.claude/agents/content-filter.md` — defines the filtering rules as a standalone agent spec

## GitHub Pages

- `docs/index.html` — single-page static site that renders `docs/digest-latest.md` using marked.js (CDN)
- `docs/digest-latest.md` — copy of today's digest, updated by GitHub Actions
- `.github/workflows/daily-digest.yml` — cron at 02:00 UTC (09:00 ICT), generates digest, copies to docs/, commits
- Pages serves from `docs/` on `main` branch

## Rules

- No API keys in code — the HN API requires none. Use `process.env` if ever needed.
- Output files in `output/` are generated — never edit them manually.
- `docs/digest-latest.md` is generated by the workflow — never edit manually.
- `generate.js` mirrors the skill and agent definitions. If you change filtering rules, update both `filterStories()` and `content-filter.md`. If you change the output format, update both `formatDigest()` and `SKILL.md`.
